# revüe v1.1.0 — Acceptance Report

Each v1.1 guarantee, mapped to the eval case(s) that prove it. Reproduce everything with one command:

```bash
python3 scripts/run-evals.py   # 102/102 must pass
```

**Provenance note.** The referenced build spec (`revue-v1.0-build-handoff.md`) and the named craft
exemplars (`vip-flagship-page.html`, `vip-instance-real.html`) were not present in the repository or
the session's uploads during either v1.1 pass — the same situation both v1.0 passes documented (see
`docs/v1.0-acceptance-report.md`). The Premium craft rules were built from the v1.1 task's own
distilled description, and the three failure classes below are taken verbatim from the v1.1 finishing
task's enumeration, which is treated as authoritative. `examples/premium-exemplar.html` is a
reconstruction built to satisfy every elevate rule, not a copy of an unseen original — if the real
flagship surfaces, diff against it.

## Guarantee 1 — The tier bar: a Premium declaration must be earned

A page declared Premium that actually ships clean-template (no display type system, no full-bleed
imagery, no legibility scrim, no persistent action bar) FAILS its tier — even when its colors, fonts,
Hard NOs, and structure are all clean.

Proving evals:
`red team: Premium-declared clean-template FAILS its tier (base audit clean, elevatePass false)` ·
`red team: clean-template misses all four elevate rules by name` ·
`red team: ship over the clean-template Premium audit REJECTED as Standard-bar-only` ·
`reject: Premium ship requires elevatePass — meets only the Standard bar` ·
`reject: Premium ship with no elevate checks run at all` ·
`dogfood-premium: clean-template draft fails ALL FOUR elevate checks while base audit is clean` ·
`dogfood-premium: 'ship' over the clean-template draft audit REJECTED as Standard-bar-only`.

The bar resists gaming, not just absence:
`red team: gamed elevate heuristics (hidden hero, sticky masthead, two-sans) all FAIL` (a hidden div
with "hero" in its class + a 1px image + a stray text-shadow, a sticky TOP masthead, and two sans
fonts each fail their check) ·
`red team: bare elevatePass with no elevateChecks REJECTED as hand-asserted` ·
`red team: elevatePass true over a failing elevate check REJECTED as inconsistent`.

And it stays a bar, not a ban: `premium exemplar: all four elevate checks pass` ·
`dogfood-premium: flipped to 'ship' with the real passing audits, validates` ·
`standard tier: same deliverable passes clean with no elevateChecks` (Standard work is never held to
a bar it wasn't briefed for) · `accept: Custom tier ship with a proper human tierSignoff` — while
`red team: Custom tierSignoff.by = 'N/A' placeholder REJECTED` keeps Custom genuinely human-signed.

## Guarantee 2 — Conformance: the brief's structure is law

A deliverable that ignores the brief's structure — a banned custom header, missing/reordered required
sections, hardcoded content where editable placeholders are required — FAILS conformance, and
disguising the violation does not evade the check.

Proving evals:
`conformance: missing section caught` · `conformance: order violation caught` ·
`conformance: forbidden element (banned custom header) caught` ·
`conformance: missing required placeholder caught (guessed value instead)` ·
`red team: reformatted banned header (case/quotes/multi-class) still caught` ·
`red team: empty stub faking a required section caught` ·
`red team: reordered required sections caught` ·
`red team: placeholder hidden while a guessed value ships caught` ·
`red team: conformance pass flipped true over listed violations REJECTED` ·
`reject: declared brief.structure but conformance fails` ·
`reject: declared brief.structure but conformance never ran`.

Precision guards (the gate must not cry wolf):
`conformance: sections present, ordered, no forbidden elements, placeholder intact -> pass` ·
`conformance: premium exemplar's real sections conform to its own structure spec` ·
`dogfood-premium: the same draft CONFORMS structurally — only the tier gate catches it` (conformance
and tier are independent gates; one does not stand in for the other).

## Guarantee 3 — Brand: the original failure stays impossible

The original off-palette generic clone (guessed template colors, off-brand font, generic link text,
fabricated "4.9 / 200+" metric) still FAILS the brand audit, with the full v1.0 evasion coverage
intact.

Proving evals:
`red team: original off-palette generic clone still FAILS the brand audit` (the explicit v1.1
re-assertion), plus the unchanged v1.0 battery — the five `red team: original failure — ...` cases and
the eight evasion fixtures (CSS-variable colors, hsl/rgb near-misses, hash-less attribute hex,
data-URI/atob payloads, external stylesheets, isolated fabricated metrics, off-lock fonts,
homoglyph/zero-width/entity/tag-split Hard NOs).

Precision improved this pass:
`regression: color-word var() names no longer false-positive as named colors` (an in-lock deliverable
using `--navy`/`--coral` custom properties passes) while
`red team: off-lock named color in a var() fallback still caught` keeps the closure airtight.

## End-to-end dogfood

`examples/dogfood-premium/` runs the whole v1.1 cascade on real work: a Premium brief with a structure
spec, `deep`-tier concept generation, a clean-template draft caught by the tier gate alone (brand and
conformance both passed it — the precise failure class this release exists to stop), the rebuilt
winner passing all gates, and a final `ship with changes` verdict with the browser check owner-tagged.
Seven `dogfood-premium: ...` evals keep it reproducible. The v1.0 dogfood (`examples/dogfood/`)
remains green and untouched as the Standard-tier exemplar.
