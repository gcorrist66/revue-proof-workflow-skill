# revüe v1.0.0 — Acceptance Report

Each of the six acceptance criteria, mapped to the eval case(s) that prove it. Reproduce everything
with one command:

```bash
python3 scripts/run-evals.py   # 58/58 must pass
```

**Provenance note.** The referenced build spec file (`revue-v1.0-build-handoff.md`) was not present in
the repository or the session's uploads during either build pass (the Sonnet pass documented the same
absence in `HANDOFF-TO-FABLE.md`). The six criteria below are taken verbatim from the v1.0 finishing
task's own enumeration, which is treated as authoritative.

## Criterion 1 — A generic clone is rejected

The exact original failure is a committed fixture: `examples/redteam-original-failure.html`, a
template-clone landing page with guessed off-palette colors (`#1a73e8`, `#34a853`, ...), an off-brand
template font (Inter), generic link text ("click here"), and a fabricated "4.9 / 200+" metric.

Proving evals:
`red team: original failure — generic clone REJECTED` ·
`red team: original failure — guessed template colors caught` ·
`red team: original failure — fabricated 4.9 / 200+ metric caught` ·
`red team: original failure — off-brand template font caught` ·
`red team: original failure — generic 'click here' caught`

Eight further red-team fixtures prove the audit resists active evasion (CSS-variable colors, hsl/rgb
near-misses, hash-less attribute hex, data-URI and atob() payloads, external stylesheets, isolated
fabricated metrics, off-lock fonts, homoglyph/zero-width/entity/tag-split Hard NOs) — all
`red team: ...` cases. The gates on the run artifact are also red-teamed: an undeclared creative run,
a forged audit object, a paraphrased duplicate concept, and a stringly-typed `handoffPageRead` are all
rejected.

## Criterion 2 — No brief blocks with a batched field request

`scripts/validate-run.py` rejects an incomplete brief with **one** failure line naming every missing
field at once.

Proving evals:
`reject: incomplete brief batches every missing field in one failure` ·
`dogfood: incomplete brief blocked with one batched field request` (asserts exactly one batched line
on the real dogfood run `examples/dogfood/run-incomplete-brief.json`).

## Criterion 3 — Single-option generation is rejected

Creative generation must present 2–3 distinct, lock-compliant concepts.

Proving evals:
`reject: fewer than 2 options for creative generation` (1 option) ·
`reject: more than 3 options for creative generation` ·
`reject: options are not actually distinct` ·
`red team: paraphrased duplicate distinctionAxis (case/punctuation) REJECTED`.

## Criterion 4 — Ship without `outputAudit.pass` is rejected

Under `--strict`, a `ship` verdict on a creative-production run requires `outputAudit.pass == true`,
and a forged audit (pass flipped true over listed violations) is rejected as internally inconsistent.

Proving evals:
`reject: ship verdict with a failing output audit` ·
`reject: ship verdict with no output audit at all` ·
`red team: forged outputAudit (pass flipped true over listed violations) REJECTED` ·
`red team: creative run undeclared as creative-production REJECTED` (a run cannot dodge the gate by
not declaring itself).

## Criterion 5 — Model routing present; gate steps tagged fast

`references/model-routing.md` maps every pipeline step to a tier (fast = Haiku 4.5, standard =
Sonnet 5, deep = Fable 5, deep-coding = Opus 4.8); `scripts/validate-run.py` enforces that known
gate/validator steps are tagged `fast` and that tiers are from the known enum.

Proving evals:
`reject: gate step (brief-gate) not tagged fast` ·
`reject: unknown modelTier value` ·
`golden worked-creative-production.json validates (strict)` and `dogfood: full run artifact validates
(strict)` (both goldens carry `fast`-tagged gate steps in `trace`).

## Criterion 6 — All prior 20 evals still green

The pre-v1.0 suite (8 golden-JSON checks, 4 golden-Markdown checks, 6 failure-mode rejections, 2
regression guards = 20 cases) runs unmodified inside the current suite and passes: the
`golden sample-run-vip-lake-travis.json ...` through `reject: path-to-ship item with a bad owner`
cases plus both `regression: ...` cases. Current total: **58/58** (20 pre-v1.0 + 14 v1.0 pillar cases
+ 19 red-team cases + 5 dogfood cases).

## End-to-end dogfood

`examples/dogfood/` is a real pipeline run on revüe's own brand: brief gate blocked first (batched),
three distinct concepts, keep/kill/change feedback, a draft rejected by the audit for a genuinely
guessed hover shade (`#e64435`, labeled near-miss of `#e5484d`), the fixed winner audited clean, and a
final `ship with changes` verdict with an owner-tagged path to ship. See `examples/dogfood/README.md`.
